# 本地完整测试计划

## 阶段一：服务启动与基础验证（预计 30 分钟）

### 1.1 确认服务运行状态

- [ ] 检查 8000 端口是否被占用
- [ ] 如果服务未运行，使用 `run.bat` 或手动命令启动
- [ ] 访问 `http://localhost:8000/health` 确认服务健康
- [ ] 访问 `http://localhost:8000/docs` 查看 API 文档

### 1.2 管理后台功能测试

- [ ] 访问 `http://localhost:8000/admin`
- [ ] 输入 Token（默认 `dev-token`）
- [ ] 检查统计卡片是否正常显示

### 1.3 数据库初始化检查

- [ ] 确认 `data/license.db` 文件已创建
- [ ] 检查数据库表结构是否完整（cards, licenses, redeem_logs, license_logs）

## 阶段二：卡密生成与管理测试（预计 30 分钟）

### 2.1 生成卡密

- [ ] 选择月卡（30 天），生成 5 个，记录卡密
- [ ] 选择季卡（90 天），生成 3 个，记录卡密
- [ ] 选择年卡（365 天），生成 2 个，记录卡密
- [ ] 检查是否自动填充正确天数（30/90/365）

### 2.2 验证卡密列表

- [ ] 切换到"未使用"状态筛选，确认刚生成的卡密显示正确
- [ ] 检查卡密类型、天数是否显示正确
- [ ] 测试"来源"筛选功能

### 2.3 作废卡密测试

- [ ] 选择一个未使用的卡密，点击"作废"
- [ ] 刷新列表，确认状态变为 `disabled`
- [ ] 尝试用已作废卡密兑换（应失败）

## 阶段三：卡密兑换流程测试（预计 1 小时）

### 3.1 准备测试账号

- [ ] 准备 3 个测试用的影刀账号（可以用测试数据）
- [ ] 账号命名建议：`test_user_001`, `test_user_002`, `test_user_003`

### 3.2 正常兑换测试

- [ ] 使用 `test_user_001` 兑换一个月卡卡密
- [ ] 检查返回结果：状态应为 `active`，剩余天数应为 30
- [ ] 在管理后台检查授权列表，确认账号已出现

### 3.3 重复兑换测试

- [ ] 使用 `test_user_001` 再次兑换一个季卡卡密
- [ ] 检查授权是否正确累加（30+90=120 天）
- [ ] 在管理后台确认到期时间已延长

### 3.4 异常兑换测试

- [ ] 使用不存在的卡密兑换（应提示"卡密不存在"）
- [ ] 使用已作废的卡密兑换（应提示"卡密已作废"）
- [ ] 使用已使用的卡密兑换（应提示"卡密已被使用"）

### 3.5 检查日志

- [ ] 在管理后台查看"兑换日志"
- [ ] 确认成功/失败的兑换都有记录
- [ ] 检查日志内容是否完整

## 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 服务无法启动 | 端口被占用 | `netstat -ano | findstr :8000` 查找并结束进程 |
| 卡密兑换失败 | 数据库未初始化 | 删除 `data/license.db` 重启服务 |
| 影刀模块导入失败 | 缺少 requests | `pip install requests` |
| 心跳失败 | 网络不通 | 检查防火墙，确认服务运行 |
| Token 验证失败 | Token 不匹配 | 检查 `.env` 中的 `ADMIN_TOKEN` |
